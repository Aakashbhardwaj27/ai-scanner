ai-scanner

Scan your codebase for LLM SDK usage, AI framework integrations, and exposed API tokens.

Zero dependencies. Works with Node.js 18+. Detects 16+ LLM SDKs, 20+ AI frameworks, and 20+ API token patterns across Python, JavaScript, TypeScript, Go, Java, and more.

🚨 Token Detection

Catches hardcoded API keys for OpenAI, Anthropic, Google, AWS, HuggingFace, Groq, and 15+ other providers.

📦 SDK Discovery

Detects imports and client initialization for 16+ LLM SDKs across all major languages.

🧠 Framework Detection

Finds LangChain, LlamaIndex, CrewAI, AutoGen, DSPy, Vercel AI SDK, and 20+ more.

🧹 Smart Filtering

Context-aware — ignores mentions in docs and README files. Skips .env files by default.

⚡ Zero Dependencies

Pure Node.js. Runs instantly with npx. No install needed.

🔄 CI/CD Ready

JSON and SARIF output. Exit code flag. GitHub Actions, pre-commit hooks, scheduled audits.

Installation

# Run directly — no install needed
npx ai-scanner ./my-project

# Or install globally
npm install -g ai-scanner
ai-scanner ./my-project
Tip

Requires Node.js 18 or higher. No external dependencies — the tool is completely self-contained.

Quick Start

Point ai-scanner at any directory to get instant results:

# Scan current directory
ai-scanner

# Scan a specific project
ai-scanner ./my-project

# Security-only: just find exposed tokens
ai-scanner --tokens-only

CLI Options

FlagDescription
-o, --output <file>Write JSON report to file
--sarif <file>Write SARIF report (for CI/CD integration)
--tokens-onlyOnly scan for exposed tokens (security mode)
--scan-envInclude .env files (skipped by default)
--no-endpointsSkip API endpoint detection
--no-modelsSkip model name reference detection
--jsonOutput results as JSON to stdout
--exit-codeExit with code 1 if critical/high findings
-h, --helpShow help message
-v, --versionShow version number
 
# Combine flags for CI pipelines
ai-scanner ./src --tokens-only --exit-code --json

# Generate SARIF for GitHub code scanning
ai-scanner --sarif results.sarif --exit-code

# Include .env files in the scan
ai-scanner --scan-env

Smart Filtering

ai-scanner is context-aware. It distinguishes between using an SDK in source code vs. mentioning it in documentation.

File typeSDK/Framework mentionsExposed tokens
Source code (.js, .py, .go, etc.)✅ Reported✅ Reported
README, docs, markdown❌ Ignored✅ Reported
examples/, samples/, docs/ dirs❌ Ignored✅ Reported
.env files❌ Skipped❌ Skipped by default
.env with --scan-env—✅ Reported
Why skip .env?

.env files are designed to hold secrets — they should be in your .gitignore. The real risk is keys hardcoded directly in source files. Use --scan-env if you want to include them.

Examples

Scan a GitHub Repo

Clone any public repo and scan it in one command:

git clone --depth 1 https://github.com/user/repo /tmp/repo
npx ai-scanner /tmp/repo
rm -rf /tmp/repo

Or use the included helper scripts:

# Shell — clones, scans, cleans up automatically
./examples/scan-github-repo.sh https://github.com/langchain-ai/langchainjs

# Node.js — uses ai-scanner as a library, saves JSON report
node examples/scan-github-repo.js https://github.com/langchain-ai/langchainjs

Batch Scan Multiple Repos

Audit multiple repos at once and get a summary table:

# Edit the REPOS array in the script, then run:
node examples/scan-multiple-repos.js

# Output:
#   🚨 my-api          12 findings   3 critical  2 SDKs  1 frameworks
#   📦 my-app           8 findings   0 critical  3 SDKs  2 frameworks
#   ✅ my-lib           0 findings   0 critical  0 SDKs  0 frameworks

Use as a Library

Import the scanner in your own Node.js code:

const { Scanner } = require('ai-scanner');

const scanner = new Scanner({ rootDir: './my-project' });
const result = scanner.scan();

// Access structured results
console.log(result.stats.criticalFindings);
console.log(result.findings.filter(f => f.type === 'token'));

CI/CD Integration

GitHub Actions

# .github/workflows/ai-scan.yml
name: AI Security Scan
on: [push, pull_request]
jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Scan for exposed AI tokens
        run: npx ai-scanner --tokens-only --exit-code

Pre-commit Hook

# .husky/pre-commit
npx ai-scanner --tokens-only --exit-code
Exit codes

With --exit-code, the tool returns 0 if clean, 1 if critical/high findings are detected — perfect for CI gates.
