Features

Everything you need to audit AI usage

One command gives you complete visibility into what AI tools your codebase depends on — and whether any secrets are leaking.

🚨

Token Exposure Detection

Catches hardcoded API keys for OpenAI, Anthropic, Google, AWS, HuggingFace, Groq, Replicate, and 15+ other providers. Tokens are auto-masked in output.

📦

LLM SDK Discovery

Detects imports and client initialization for 16+ LLM provider SDKs across Python, JavaScript, TypeScript, Go, Java, and more.

🧠

AI Framework Detection

Finds LangChain, LlamaIndex, CrewAI, AutoGen, Vercel AI SDK, DSPy, Haystack, Semantic Kernel, and 20+ other AI frameworks.

🧹

Smart Filtering

Context-aware scanning ignores SDK mentions in READMEs and docs folders. Skips .env files by default. Zero false-positive noise.

⚡

Zero Dependencies

Pure Node.js — no npm install needed. Runs instantly with npx. Works on Node 18, 20, and 22 across Linux, macOS, and Windows.

🔄

CI/CD Ready

JSON and SARIF output formats. Exit code flag for pipeline gates. Works as a GitHub Action, pre-commit hook, or scheduled audit.

Smart Filtering

Context-aware, not just pattern matching

ai-scanner knows the difference between using an SDK and just talking about it in your README.

File type SDK / Framework mentions Exposed tokens
Source code (.js, .py, .go, etc.) ✅ Reported ✅ Reported
README, docs, markdown ❌ Ignored ✅ Reported
examples/, samples/, docs/ dirs ❌ Ignored ✅ Reported
.env files ❌ Skipped ❌ Skipped by default
.env with --scan-env — ✅ Reported
Integration

Drop into any pipeline

GitHub Actions, pre-commit hooks, scheduled audits — ai-scanner fits anywhere.

# .github/workflows/ai-scan.yml name: AI Security Scan on: [push, pull_request] jobs: scan: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Scan for exposed AI tokens run: npx ai-scanner --tokens-only --exit-code
